# 8-bit Timers 0..3

`core/timers.py` · tests `tests/test_timers.py` · landed 2026-07-10 (DEVLOG pass 186)

Sources: Toshiba TMP95C061 datasheet + the official SNK SDK
(`01_SDK/docs/8Bit.txt`). Nothing here is guessed.

## 1. Registers

| Addr | Register | Contents |
|------|----------|----------|
| `0x0020` | `TRUN`   | bit *n* = run timer *n*; **bit 7 = PRRUN** (prescaler run) |
| `0x0022` | `TREG0`  | timer 0 compare value |
| `0x0023` | `TREG1`  | timer 1 compare value |
| `0x0024` | `T01MOD` | bits 1-0 = timer 0 clock source; bits 3-2 = timer 1 |
| `0x0026` | `TREG2`  | timer 2 compare value |
| `0x0027` | `TREG3`  | timer 3 compare value |
| `0x0028` | `T23MOD` | bits 1-0 = timer 2 clock source; bits 3-2 = timer 3 |

The TREG registers are **write-only on hardware**; we read the last value software
wrote (which is what the overlay holds). The up-counters are internal chip state
and are **not** memory-mapped — they live in the controller.

`PRRUN = 0` *"zero clears and stops the prescaler"* (SDK), so with it clear the
internal taps produce nothing at all.

## 2. Clock sources

| mode | timer 0 / timer 2 | timer 1 / timer 3 |
|:----:|-------------------|-------------------|
| `00` | external **TI0**  | overflow of the paired timer (**16-bit cascade**) |
| `01` | `T1`              | `T1` |
| `10` | `T4`              | `T16` |
| `11` | `T16`             | `T256` |

> **On NGPC the external clock TI0 is the video H-blank.** SDK `8Bit.txt`: *"In
> the NEOGEO POCKET, the horizontal blanking interrupt generated by the 2D
> graphic controller is the external clock TI0."* So **timer 0 in mode `00`
> counts scanlines**.
>
> *"Timer 0 has an external clock TI0 terminal. Timer 2 does not."* — so mode `00`
> on timer 2 counts nothing.

## 3. Prescaler taps

The SDK gives the **measured periods** (`8Bit.txt`, Figure 2 Prescaler):
T1 = 20.83 µs, T4 = 83.33 µs, T16 = 333.3 µs, T256 = 5.333 ms.

At the NGPC's **6.144 MHz** CPU clock that is:

| tap | CPU cycles |
|-----|-----------:|
| `T1`   | **128** |
| `T4`   | **512** |
| `T16`  | **2048** |
| `T256` | **32768** |

Self-consistent: `128 × 256 = 32768`.

> **Divergence from a widely-circulated figure, resolved in favour of the SDK.**
> A base rate of `240` circulates for this (so T1 = 240 cycles, ~1.875× slower),
> but it is a hand-tuned constant from code that admits as much in its own
> comments. The SDK states *measured periods*; a tuned constant does not
> outrank that.

## 4. Comparator

SDK `8Bit.txt` § 4: *"The comparator compares the up counter value and timer
register value. If it is in unison (the same), the up counter is **0 cleared** and
an **interrupt (INTT0~3)** is generated."*

*"If the value is set to 00H, unison check becomes active when the up counter is
in **overflow** status"* — so `TREG = 0` means **256 ticks**, not zero.

## 5. Interrupts

Toshiba **Table 3.3 (1)** (hardware vector index = vector value / 4):

| source | vector | address | index | HDMA |
|--------|--------|---------|------:|------|
| `INTT0` — 8-bit timer 0 | `0x0040` | `0xFFFF40` | **16** | `10H` |
| `INTT1` — 8-bit timer 1 | `0x0044` | `0xFFFF44` | **17** | `11H` |
| `INTT2` — 8-bit timer 2 | `0x0048` | `0xFFFF48` | **18** | `12H` |
| `INTT3` — 8-bit timer 3 | `0x004C` | `0xFFFF4C` | **19** | `13H` |

Priority levels are **programmable**, read from the INTxx registers at delivery
time: timer 0 → `0x73` low nibble, timer 1 → `0x73` high, timer 2 → `0x74` low,
timer 3 → `0x74` high. **These are exactly the nibbles the BIOS call
`VECT_INTLVSET` writes** (see `specs/BIOS_HLE.md`) — the pieces interlock.

Level `0` means the source is disabled.

## 6. Model

`TimerController` is owned by `EmulatorSession` (the up-counters must survive
across step batches) and ticked in the run loop:

```python
raised: list[int] = timers.tick(cycles, memory)   # hardware vector indices
```

Each raised index is folded into `IrqState.pending_vectors`; delivery is the
shared `try_deliver_pending_vector_irq` path (see `specs/FRAME_TIMING.md`).

## 7. Not modelled

- The timer flip-flops (TFF1/TFF3) and their output pins.
- `TFFCR` / `TRDC` (the TREG double-buffer control).
- 16-bit timers 4/5 (`INTTR4..7`).
